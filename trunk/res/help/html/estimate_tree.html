Estimating Phylogenetic Trees

Phylogenetic trees can be created from any selected partition, and from any set of selected sequences (where the number of sequences is greater than three).

You will find the different methods for creating phylogenetic trees in the Analysis menu under section Phylogeny

Neighbour Joining Methods

There is currently one method for estimating NJ trees:

Quick Tree Estimation

This method allows you to get a quick, rough tree. It uses F84+Gamma model (with fixed ts/tv and alpha parameter) for DNA alignments and the WAG model for protein alignments.

Bayesian Tree Estimation

Bayesian Tree Estimation is done using

MrBayes

MrBayes is probably the most sophisticated tree estimation, but is rather slow on larger datasets.
There are two ways you can launch a MrBayes job:

  • One Model: MrBayes will then use one substitution model for the whole alignments
  • Codon Position Models: MrBayes will use a separate model for each codon position

You are able to specify the model(s) you want to use and general MrBayes parameters on the 'Advanced' tab.

Maximum Likelihood Tree Estimation

Using a maximum likelihood method is a more sophisticated way than using Neighour Joining and is still reasonable quick for tree estimation. It can be used for larger datasets.
There are currently two methods available:

PhyML

On the PhyML dialog you can specify the substitution model you want to use and general options for the PhyML program.

RaxML

Like MrBayes you can run RaxML either with one model for the whole alignment or separated models for each codon position. Bootstrapping will run in parallel, so RaxML is a good choice for bootstrapped tree estimation on large datasets.

On the 'Advanced' tab you can specify the substitution model you want to use and general RaxML parameters.
The substitution model options are different from the other methods:
For rate heterogenity you have three choices:
  • CAT: This will use a (faster) gamma approximation for rate heterogenity
  • MIX: Uses CAT to perform tree inference and GAMMA for the final tree evaluation
  • GAMMA: Uses the common gamma model with 4 rate categories
Note that for DNA alignments the only substition model available is GTR.
