Estimating Phylogenetic Trees

Phylogenetic trees can be created from any selected partition, and from any set of selected sequences (where the number of sequences is greater than three).

To create a phylogenetic tree, select Analysis | Estimate Phylogenetic Tree from the menu bar. This brings up the Estimate New Tree dialog.

F84+Gamma

The F84+Gamma model (with fixed ts/tv and alpha parameter) can be used for a quick, rough tree estimation. It has no further options.

PhyML

Using a maximum likelihood method, like PhyML is a more sophisticated but still reasonable quick way for tree estimation. It can be used for larger datasets.
On the 'Advanced' tab you can specify the substitution model and further PhyML specific parameters.

MrBayes

MrBayes is probably the most sophisticated tree estimation method at the moment, but is rather slow on larger datasets.
Like for the maximum likelihood method you can specify the substitution model and further MrBayes parameters on the 'Advanced' tab.

MrBayes (cDNA)

This method also uses MrBayes but with a cDNA model. Choosing this method, MrBayes will estimate a model for each codon position.
