Analysis Settings

The Analysis Settings dialog contains globally-applied settings that relate to various options used by TOPALi while running analyses.

Web/Cluster:

  • TOPALi web services - holds the location of the remote TOPALi analysis server that jobs will be submitted to.
  • Check job progress every - specifies the time in seconds between status checks of jobs' progress (both local and remote).
  • Check for updated versions of TOPALi at program startup - if selected, will cause TOPALi to connect to the internet and query for newer versions each time the program is loaded.
  • Use a proxy server for all web connections - if selected, ensures that all internet requests made by TOPALi are redirected via the specified proxy server. If NTLM (domain) authentication is required, it can be used by entering the username in the form "DOMAIN\username" into the Username field.

Local Job Settings:

  • TOPALi temporary working directory - lists how much disk space TOPALi has used for running local jobs. The files can be deleted by pressing the Clear button.
  • Maximum number of CPU/Cores to use for local jobs - specifies how many concurrent threads will be run together for local jobs. A higher number will result in faster job completion (for analysis jobs that can be run in parallel) but will leave little resource for other non TOPALi tasks your PC is running.
Note that neither of the above two settings are available for modification if TOPALi is currently running locally submitted jobs.
