Likelihood Ratio Test

A common approach to comparing trees estimated from 2 alignments (that contain the same "species")
is to carry out a Likelihood Ratio Test.
The idea is that you compare 2 hyptheses:
(A) the same tree for both alignments and
(B) Different trees for both alignments.


The test statistic is the difference in the Log Likelihood (or 2 times the difference in the Log Likelihood
as this is distributed as a Chi squared distribution). The Log Likihood is usually obtained by doing
a standard Maximum Likelihood (ML) phylogenetic analysis of each alignment.


We do an approximate ML analysis by estimating a Neighbor Joining tree (NJ)
(using optimised parameters for the F84 nucleotide substitution model) and computing
the Log Likelihod for this NJ tree. So instead of using a heuristic search method
(like NNI or SPR or TBR in PAUP) we just use NJ as the search method.
This is therefore faster but approximate.


This means we can move a window along the alignment, just like the DSS method,
but do a fast LRT test between the 2 half-windows.
Like DSS, we use parametric bootstrapping (simulating alignments with no recombination,
and then estimating a significant threshold) to test statistical significance of peaks
produced bt the LRT method.
