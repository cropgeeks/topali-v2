Positive selection analysis using the PAML package


The variability in each column of a protein-coding alignment contains information that can be used to investigate the process of protein evolution.  Evolutionary change occurs through mutation (e.g. substitution of nucleotides) followed by a process called natural selection which may remove some of the changes (e.g. substitutions), or do nothing, or amplify the frequency of some of the changes.  If there is no selection we would expect a rate of evolution that reflects the rate of nucleotide substitution (i.e. the neutral rate).  If the rate (at a particular site) is significantly lower than the neutral rate, we might infer that selection has removed substitutions that result in a less functional protein.  Alternatively, if we see higher than the neutral rate at a site, we might infer that this site is under positive selection as it improves protein function.  With protein-coding DNA, the underlying structure of the Genetic Code (i.e. the mapping of 61 sense codons to the 20 amino acids) allows us to distinguish nucleotide changes that affect or do not affect amino acid choice.  ASSUMING that the changes that do not affect amino acid change are neutral (this assumption may not always be completely true) allows us to use these "synonymous" changes as a standard against which to compare the nonsynonymous (i.e. amino acid altering) changes.

The analysis of positive selection requires an understanding of molecular evolution and of the appropriate software.  TOPALi uses the PAML package (Phylogenetic Analysis by Maximum Likelihood) by Ziheng Yang for Selection Analysis.  See the PAML webpage (see below)  and the 2007 PAML overview (see below) for more information on PAML.  A 2005 PAML book chapter (plus associated online PAML example datasets and online PAML demo material) provides excellent directions for  analysing real data.   TOPALi uses only  the codeml program from PAML, so there is no need to read up on other PAML programs.

TOPALi can launch two types of web service  to analyse  protein-coding  DNA alignments for detect positive selection, which we call the codeml sites model and codeml branch model.    These two types of analysis are discussed in the PAML book chapter and the PAML demo material.   See TOPALi documentation on the codeml sites model settings and codeml branch model settings for more detail.

In this version of TOPALi, one aim has been to simplify the use of PAML for users already familiar with the PAML package.  They no longer need to worry about PAML ctl files, for example.   For users new to positive selection analysis, we strongly suggest  you read the  PAML book chapter and the PAML demo material first and try to replicate the analysis of the PAML example datasets using TOPALi.  After that, you should be ready to analyse your own data.
